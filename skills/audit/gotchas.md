# Audit Gotchas

## 자주 발생하는 실수

### 1. 커밋 메시지만 보고 코드 안 읽음
❌ `git log`의 커밋 메시지만 확인하고 "변경사항 확인 완료"
→ ✅ `git diff --staged`로 실제 코드 변경 내용을 한 줄씩 읽음

커밋 메시지가 "fix: 버그 수정"이어도 실제 코드에 보안 취약점이 포함될 수 있다.

### 2. Staged 파일만 검사하고 Unstaged 누락
❌ `git diff --staged`만 확인하고 `git diff` (unstaged) 무시
→ ✅ staged + unstaged 모두 확인. `git status`로 전체 상태 파악 후 검사

unstaged 변경이 의도치 않게 누락되면 불완전한 코드가 커밋된다.

### 3. 보안 검사 스킵
❌ 기능 코드만 리뷰하고 하드코딩 시크릿, SQL injection, XSS 검사 건너뜀
→ ✅ 모든 변경 파일에 대해 보안 패턴 스캔 수행

`.env` 파일 커밋 여부, API 키 하드코딩, raw SQL 사용을 반드시 체크하라.

### 4. 새로 추가된 파일 무시
❌ 수정된 파일만 검사하고 새로 추가된(untracked) 파일 검토 안 함
→ ✅ `git status`에서 new file도 반드시 내용 확인

새 파일에 테스트 없는 코드, 불필요한 의존성, 시크릿이 포함될 수 있다.

### 5. Conventional Commit 형식 미검증
❌ 커밋 메시지 내용만 보고 형식(`feat:`, `fix:`, `refactor:` 등) 검증 안 함
→ ✅ 커밋 메시지가 Conventional Commits 규격을 따르는지 확인

형식이 맞아야 자동 변경 로그 생성과 시맨틱 버저닝이 정상 동작한다.

### 6. 삭제된 코드의 영향 미확인
❌ 코드 삭제를 "정리"로만 보고 참조하는 곳이 있는지 확인 안 함
→ ✅ 삭제된 함수/클래스/상수를 import하는 파일이 없는지 검색

삭제 후 import 에러가 런타임에서야 발견되면 장애로 이어진다.
