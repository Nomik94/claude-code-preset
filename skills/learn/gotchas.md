# Learn Gotchas

## 자주 발생하는 실수

### 1. 일시적 버그를 영구 교훈으로 저장
❌ 오타나 단순 실수를 "교훈"으로 memory에 영구 저장
→ ✅ 반복 가능성이 있는 패턴/설계 문제만 교훈으로 기록

"변수명 오타 주의"는 교훈이 아니다. "lazy='raise' 설정 시 joinedload 필수"가 교훈이다.

### 2. 너무 구체적인 해결책 기록 (범용성 없음)
❌ "user_service.py 42번째 줄에서 await 빠뜨림" — 다음에 재사용 불가
→ ✅ "async 함수에서 sync I/O 호출 시 이벤트 루프 블로킹 발생" — 패턴 수준으로 추상화

교훈은 특정 파일이 아닌 패턴에 대한 것이어야 다른 상황에서도 적용 가능하다.

### 3. 해결 방법만 기록하고 원인 누락
❌ "time.sleep을 asyncio.sleep으로 바꿈" — 왜?
→ ✅ "원인: async 함수 내 sync 블로킹이 이벤트 루프를 멈춤. 해결: asyncio.sleep 사용"

원인 없는 해결책은 비슷하지만 다른 상황에 잘못 적용될 수 있다.

### 4. 중복 교훈 축적
❌ 같은 패턴에 대한 교훈이 memory에 3-4개 중복 저장
→ ✅ 새 교훈 저장 전에 기존 교훈 확인. 중복 시 기존 항목 보강

중복 교훈은 노이즈를 늘리고 정작 중요한 교훈의 가시성을 떨어뜨린다.

### 5. 교훈을 프로젝트에 국한
❌ 특정 프로젝트에서만 유효한 교훈을 범용 교훈처럼 저장
→ ✅ 프로젝트 특화 교훈과 범용 교훈을 구분하여 저장

"이 프로젝트는 PostgreSQL 15 사용"은 교훈이 아니라 프로젝트 메타데이터다.

### 6. 교훈 저장 후 활용하지 않음
❌ memory/에 교훈을 저장만 하고 세션 시작 시 참조하지 않음
→ ✅ 세션 시작 시 관련 교훈을 로드하고 구현에 반영

저장만 하고 읽지 않는 교훈은 존재하지 않는 것과 같다.
