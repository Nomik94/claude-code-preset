# Learn

디버깅 인사이트와 프로젝트 고유 지식을 향후 세션용으로 기록.

## 저장 조건 (4개 모두 충족 필수)

| # | 조건 |
|---|------|
| 1 | 근본 원인 확인 ("왜" 메커니즘 명확) |
| 2 | 재발 가능성 있음 |
| 3 | 범용적 해결 (이 케이스에만 한정되지 않음) |
| 4 | 기존 문서 부재 (5분 내 검색 불가) |

## 저장 대상

**저장함:** 프로젝트 고유 버그 패턴, 설정 함정, 아키텍처 결정 근거, 의존성 비호환, 모듈 간 비직관적 상호작용, 성능 함정, BE/FE 통신 이슈

**저장 안 함:** 일반 프로그래밍 지식, 표준 라이브러리 사용법, 공식 문서 내용, 일회성 오타

## 저장 위치

경로: `memory/` — 파일명은 의미 있는 이름 사용 (예: `sqlalchemy-async-session-leak.md`)

## 저장 형식

```markdown
---
learned: YYYY-MM-DD
tags: [relevant, searchable, tags]
severity: high | medium | low
stack: be | fe | fullstack
---

## 증상
[에러 메시지 또는 증상. 파일 경로 포함.]

## 원인
[왜 발생했는지. 메커니즘 구체적으로.]

## 해결
[정확한 코드/설정 변경 또는 명령어.]

## 예방
[재발 방지: CI 체크, 린트 규칙, 패턴.]
```

## 품질 게이트

저장 전 모두 예여야 저장, 아니면 폐기:
- [ ] 디버깅 30분+ 절약 가능?
- [ ] 근본 원인 명확?
- [ ] 파일 경로/코드 스니펫 포함?
- [ ] 증상→원인→해결→예방 흐름 완전?

자주 발생하는 실수는 이 디렉토리의 gotchas.md를 참조하라.
