# Note Gotchas

## 자주 발생하는 실수

### 1. Priority Context 500자 초과
❌ Priority Context(`--priority`)에 긴 설명이나 코드를 저장 → 500자 초과
→ ✅ Priority Context는 500자 이내의 핵심 정보만. 상세 내용은 Working Memory 사용

Priority Context는 항상 로드되므로 크기가 크면 매번 컨텍스트를 낭비한다.

### 2. Working Memory에 코드 덩어리 저장
❌ 함수 전체나 설정 파일 전체를 `/note`로 저장
→ ✅ 파일 경로와 핵심 포인트만 기록. "코드는 /path/to/file.py 참조"

코드는 파일에 있다. 노트에 복사하면 동기화가 안 되고 컨텍스트만 낭비한다.

### 3. 타임스탬프 없이 저장
❌ 언제 저장한 노트인지 알 수 없어서 오래된 정보와 최신 정보 구분 불가
→ ✅ Working Memory는 자동 타임스탬프 포함. `/note --prune`으로 7일+ 항목 정리

오래된 노트가 쌓이면 현재 작업과 무관한 노이즈가 된다.

### 4. 구조 없이 무분별하게 저장
❌ 모든 종류의 정보를 구분 없이 Working Memory에 나열
→ ✅ 용도별 분류: `--priority` (핵심, 항상 로드), 기본 (작업 컨텍스트), `--manual` (영구)

분류 없는 노트는 나중에 필요한 정보를 찾기 어렵다.

### 5. compact 전에 노트 저장 안 함
❌ `/compact` 실행 전에 현재 작업 상태를 `/note`로 저장하지 않음
→ ✅ compact 전에 반드시 현재 진행 상황, 다음 단계, 주의사항을 노트로 저장

compact 후에는 이전 컨텍스트가 압축되므로 세부 사항을 잃을 수 있다.

### 6. MANUAL 섹션 남용
❌ 일시적 작업 정보를 `--manual`로 저장하여 영구 보관
→ ✅ `--manual`은 프로젝트 전체에 걸쳐 항상 유효한 규칙/컨벤션에만 사용

MANUAL은 영구 저장이므로 일시적 정보가 쌓이면 정리가 필요해진다.
